## Current work types to validate

During discovery, classify each activity as one of the following:

| Category | Meaning |
|---|---|
| Recurring information maintenance | Regular content, tool or knowledge update work |
| Urgent information update | Time-sensitive communication or content update |
| Ad hoc support request | Work triggered by a specific request |
| Editor support | Work that enables other people to maintain information |
| CRM or system support | Work relating to user issues, system questions or maintenance |
| Hot topic response | Work triggered by campaigns, media, public issues or operational events |
| Exception handling | Work that happens when information is missing, conflicting or unclear |
| Informal workaround | Work done manually because process, system or ownership is unclear |
| Shared responsibility | Work that depends on another team, editor, service area or system owner |
| Role gap | Work expected but not currently performed |
| Future capability signal | Work that may be required in a future operating model |

## Key questions for validation

- What are the top five activities performed in a normal week?

- What systems are updated most often?
- What requests are hardest to resolve?
- What work is known only by the officer?
- What work is expected by others but not actually performed?
