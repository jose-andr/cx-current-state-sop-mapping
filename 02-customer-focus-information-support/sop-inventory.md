# SOP Inventory

## Function

Customer Focus Information Support

## Purpose

This inventory lists current-state SOP candidates for the Customer Focus Information Support function.

The inventory should be validated with the officer who performs the work before any SOP is treated as confirmed current practice.

## Current-state mapping rule

Only map SOPs for work that is actually performed today.

If a responsibility appears in the role description but is not active in practice, capture it in `actual-practice-summary.md` as a role gap, shared responsibility or future capability signal.

## Areas requiring validation before inclusion

The following areas may be active, partial, historic or shared. Do not create SOPs until actual practice is confirmed.

| Area | Reason for validation | Where to capture if unclear |
|---|---|---|
| Daily CRM maintenance and configuration | May be system administration, user support or escalation rather than direct ownership. | `actual-practice-summary.md` |
| Customer service reporting support | May be owned by another role or only supported occasionally. | `actual-practice-summary.md` |
| External media monitoring | May be informal, historic or shared with communications teams. | `actual-practice-summary.md` |
| Customer Focus Strategy project support | May be project-based rather than recurring SOP work. | `actual-practice-summary.md` |
| CoMWeb or SharePoint publishing | Actual tools, access and update responsibilities need validation. | `systems-and-tools.md` |

## Prioritisation criteria

Use the following criteria to decide which SOPs to map first.

| Criterion | Question |
|---|---|
| Frequency | Does this happen often? |
| Urgency | Does this need to happen quickly when triggered? |
| Risk | Could outdated or incorrect information affect customers or staff? |
| Dependency | Is the work dependent on one person’s knowledge? |
| Ambiguity | Are ownership, approval or publishing rules unclear? |
| Rework | Does the activity often need correction, clarification or follow-up? |
| First contact resolution relevance | Does this activity help staff resolve enquiries without escalation? |
| Omnichannel relevance | Does this information need to be reused across channels in future? |
| Automation relevance | Could this activity reveal future knowledge reuse or workflow automation opportunities? |
