## 1. Purpose

This SOP documents the current-state process for supporting CRM user issues and requests that come to the Customer Focus Information Support function.

The purpose is to understand what types of CRM-related requests are handled today, how they are assessed, what can be resolved directly, and what requires escalation to a system owner, technology support or another function.

This SOP should be validated with the officer who performs the work before it is treated as confirmed current practice.

## 2. Scope

## In scope

- receiving CRM user issues or requests
- clarifying the nature of the issue
- determining whether the request relates to user guidance, service information, data quality, access, configuration or system error
- resolving requests that are within the function’s current practice
- providing guidance or redirecting users where appropriate
- escalating requests outside scope, authority or access
- recording outcomes or follow-up actions

## Out of scope

- formal CRM system administration
- access provisioning unless confirmed as current practice
- system configuration changes
- technical defect resolution
- enterprise service desk processes
- customer data correction unless handled through the Customer Data and Systems Support process
- broad CRM governance design

## 5. Current process

Draft process to validate:

1. Receive the CRM user issue or request.
2. Review the request and identify what the user is trying to do.
3. Clarify missing information if required.
4. Determine the type of issue:
   - user guidance
   - service information
   - data quality
   - access or permission
   - system error
   - configuration or workflow issue
   - unclear or out-of-scope request
5. Check whether the issue can be resolved using current knowledge, guidance or access.
6. Provide guidance, correction pathway or response if within scope.
7. Escalate the issue if it requires system owner, technology, data quality or manager involvement.
8. Record the outcome or follow-up action according to current practice.
9. Confirm with the requester where required.

## 12. Omnichannel implications

CRM user issues and requests can reveal gaps in the current service knowledge, data quality and system support ecosystem.

Current-state observations may inform future requirements such as:

- clearer support categories for knowledge, data and technical issues
- staff guidance for common CRM tasks
- escalation pathways for system and access issues
- reusable troubleshooting knowledge
- pattern tracking for repeated user issues
- service knowledge updates triggered by CRM support requests
- CRM design improvements to support channel continuity
