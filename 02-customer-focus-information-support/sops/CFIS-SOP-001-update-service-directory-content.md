## 1. Purpose

This SOP documents the current-state process for updating Service Directory content.

The purpose is to understand how service information updates are requested, checked, approved, published and communicated today.

This SOP should be validated with the officer who performs the work before it is treated as confirmed current practice.

## 2. Scope

## In scope

- receiving Service Directory update requests
- clarifying requested changes
- checking source information
- confirming ownership or approval
- updating Service Directory content
- recording completed updates
- escalating unclear, conflicting or high-risk information issues

## Out of scope

- final enterprise knowledge management design
- broad content strategy
- website content governance
- CRM system configuration
- chatbot or automation design
- service redesign
- approved future-state publishing workflow

## 5. Current process

Draft process to validate:

1. Receive or identify a Service Directory content update need.
2. Review the update request and identify the affected entry or topic.
3. Check whether enough information has been provided.
4. Confirm the source of the requested change.
5. Identify the service owner, editor or approver if required.
6. Review the current Service Directory content.
7. Update the content in plain, usable language.
8. Check whether the update affects other tools, pages or staff guidance.
9. Publish or submit the update according to current practice.
10. Confirm the update is complete.
11. Record the update or notify relevant stakeholders if required.
12. Escalate unclear, conflicting, urgent or high-risk updates.
