## 1. Purpose

This SOP documents the current-state process for maintaining telephone messages on hold.

The purpose is to understand how messages are planned, updated, approved, uploaded, checked and retired.

This SOP should be validated with the officer who performs the work before it is treated as confirmed current practice.

## 2. Scope

## In scope

- identifying the need for a new or updated message on hold
- preparing message content
- confirming content accuracy
- obtaining required approval
- coordinating upload to the telephone system
- confirming the message is active
- retiring outdated messages
- recording completed updates

## Out of scope

- telephone system administration unless confirmed as current practice
- contact centre telephony configuration
- campaign strategy
- brand or marketing approval framework design
- emergency communications governance
- approved future-state message workflow design

## 5. Current process

Draft process to validate:

1. Identify that a message on hold needs to be created, updated or retired.
2. Confirm the message topic and purpose.
3. Gather source information or key message points.
4. Draft or receive message wording.
5. Check that the message is accurate, clear and suitable for callers.
6. Obtain required approval.
7. Coordinate upload to the telephone system.
8. Confirm the message is active and working as expected.
9. Record the update using the current tracking method.
10. Set or confirm review, expiry or retirement timing.
11. Remove or update outdated messages when required.
