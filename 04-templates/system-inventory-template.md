# System Inventory Template

## Purpose

Use this template to capture systems, tools, repositories and informal work environments discovered during current-state SOP mapping.

The goal is to understand how work depends on systems today, which systems are authoritative, where information is duplicated, and where workflow is happening outside formal tools.

## Instructions

Complete one row for each system or tool.

Include formal systems and informal tools such as email, Teams, spreadsheets, shared folders, trackers and local notes.

Do not assume a system is a source of truth until validated.

## Dependency questions

- What work starts in this system?
- What work ends in this system?
- What information is created here?
- What information is updated here?
- What information is only checked here?
- What information is copied from this system?
- What information is copied into this system?
- What other systems depend on this system?
- What happens if the system is unavailable?
- Who can update it?
- Who approves changes?
- Who owns the information?
- What is duplicated elsewhere?
- What is hard to find?
- What is hard to trust?
- What is hard to hand over?

## Future architecture signal prompts

- Does this system contain information that could be reused across channels?
- Does this system duplicate information from another system?
- Does this system rely on manual updates from another system?
- Does this system support workflow visibility?
- Does this system create hidden work?
- Does this system require lifecycle controls such as review or expiry dates?
- Does this system support source ownership?
- Would future automation depend on this system being accurate?
- Would omnichannel service be affected if this system was unreliable?
