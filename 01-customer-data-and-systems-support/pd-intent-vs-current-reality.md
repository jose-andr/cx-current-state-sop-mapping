# PD Intent vs Current Reality

## Purpose

This file captures the difference between the formal or intended role scope and the work actually performed today by the Customer Data and Systems Support function.

The purpose is not to critique the role or the officer.

The purpose is to create a clear baseline for current-state SOP mapping and future operating model design.

## Current status

Status: Draft discovery placeholder.

This file should be validated through officer walkthroughs and manager discussion.

## Mapping principle

Do not document aspirational responsibilities as current SOPs.

If a responsibility appears in a position description but is not active in practice, capture it here as:

- role intent
- role gap
- partially operationalised work
- transferred responsibility
- future capability signal

## Summary

The broader role description appears to give the function more weight in customer experience measurement, reporting and analysis.

In practice, the current function is understood to remain closer to operational customer data and systems support.

This distinction should be tested during current-state discovery.
