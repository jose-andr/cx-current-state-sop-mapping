# SOP Inventory

## Function

Customer Data and Systems Support

## Purpose

This inventory lists current-state SOP candidates for the Customer Data and Systems Support function.

The inventory should be validated with the officer who performs the work before any SOP is treated as confirmed current practice.

## Current-state mapping rule

Only map SOPs for work that is actually performed today.

If a responsibility appears in the role description but is not active in practice, capture it in `pd-intent-vs-current-reality.md` rather than creating a current-state SOP.

## Prioritisation criteria

Use the following criteria to decide which SOPs to map first.

| Criterion | Question |
|---|---|
| Frequency | Does this happen often? |
| Risk | Could errors create customer, data, privacy or operational risk? |
| Dependency | Is the work dependent on one person’s knowledge? |
| Ambiguity | Are the rules or decision points unclear? |
| Rework | Does the activity often need correction or follow-up? |
| Omnichannel relevance | Does the activity affect customer identity, account context or future channel continuity? |
| Automation relevance | Could the activity reveal future automation or system improvement opportunities? |

## Recommended first SOPs to map

Start with:

1. `CDSS-SOP-001-identify-duplicate-customer-records.md`
2. `CDSS-SOP-003-manually-deduplicate-customer-records.md`
3. `CDSS-SOP-005-correct-customer-data-quality-issues.md`

These are likely to expose the most useful decision points, exceptions, data quality rules and future-state implications.
