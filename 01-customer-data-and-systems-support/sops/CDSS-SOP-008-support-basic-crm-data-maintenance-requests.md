## 1. Purpose

This SOP documents the current-state process for supporting basic CRM data maintenance requests.

The purpose is to understand what CRM data maintenance requests are received, how they are assessed, what actions can be completed directly, and what issues require escalation.

This SOP should be validated with the officer who performs the work before it is treated as confirmed current practice.

## 2. Scope

## In scope

- receiving basic CRM data maintenance requests
- clarifying the request
- checking whether the request relates to customer record quality or data maintenance
- confirming whether the officer can complete the request
- completing permitted CRM data maintenance actions
- recording completed work
- escalating requests outside scope, authority or system access

## Out of scope

- CRM system administration
- system configuration changes
- access provisioning
- formal technology support
- customer experience measurement or reporting
- large-scale data migration
- approved future-state workflow design

## 5. Current process

Draft process to validate:

1. Receive the CRM data maintenance request.
2. Review the request and identify what the requester is asking for.
3. Confirm whether the request relates to customer data quality, customer record maintenance or another issue.
4. Check whether enough information has been provided.
5. Open the relevant CRM or customer data system.
6. Review the related customer record or data issue.
7. Decide whether the request can be completed directly.
8. If the request is within scope and permitted, complete the required maintenance action.
9. Record the action taken using the current tracking or system note method.
10. Notify the requester if required.
11. Escalate requests that are unclear, risky, outside authority or system-constrained.

## 12. Omnichannel implications

Basic CRM data maintenance requests can reveal whether customer data ownership, request routing and system support pathways are clear enough for future omnichannel service delivery.

Current-state observations may inform future requirements such as:

- clearer request intake pathways
- standard CRM data maintenance categories
- required information for data maintenance requests
- ownership rules for customer record changes
- escalation paths for system or access issues
- privacy-safe request handling
- visible status tracking for data maintenance work
- distinction between data maintenance, system support and service knowledge requests
