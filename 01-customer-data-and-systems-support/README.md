# Customer Data and Systems Support

## Purpose

This folder captures the current-state SOP mapping for the Customer Data and Systems Support function.

The purpose is to document how the function operates today, with a focus on customer data quality, customer record maintenance, duplicate record management, data correction and basic CRM data maintenance support.

This folder should map actual practice only.

Do not create SOPs for broader reporting, measurement or analysis responsibilities unless they are confirmed as work currently performed by the officer.

## Current-state focus

The current-state focus includes:

- identifying duplicate customer records
- cleansing and simplifying customer records
- manually de-duplicating customer records
- monitoring customer record quality
- correcting customer data quality issues
- recording progress of data cleansing work
- escalating data quality or system issues
- supporting basic CRM data maintenance requests

## Excluded unless confirmed as actual practice

The following activities should not be treated as current-state SOPs unless validated with the officer as work currently performed:

- leading customer experience measurement
- producing organisation-wide CX analysis
- producing monthly Customer Service Indicator reports
- producing customer feedback and complaints reports
- producing direct customer contact channel trend reports
- preparing ad hoc CX reports
- developing specifications for new reports
- maintaining Customer Contact reporting tools
- making organisation-wide performance recommendations

If these activities appear in the position description but are not performed in practice, capture them in `pd-intent-vs-current-reality.md`.

## Mapping questions

Use these questions when mapping the function:

- What work lands on the officer’s desk in a normal week?
- What customer data quality activities happen regularly?
- What triggers duplicate record investigation?
- What systems or tools are used to identify duplicate records?
- What rules guide cleansing, correction or de-duplication?
- What work is manual?
- What work is undocumented?
- What information is needed before a record can be corrected?
- What cases require escalation?
- What quality checks are performed?
- How is completed work recorded?
- What data quality problems keep recurring?
- What upstream processes or channels create data issues?
- What customer context would need to be reliable in a future omnichannel model?
