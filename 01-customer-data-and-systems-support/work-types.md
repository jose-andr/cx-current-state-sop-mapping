# Work Types

## Function

Customer Data and Systems Support

## Purpose

This file categorises the types of work performed by the Customer Data and Systems Support function.

The aim is to understand the shape of the work before drafting or validating individual SOPs.

## Current status

Status: Draft discovery placeholder.

Work types should be validated with the officer who performs the function.

## Work type categories

| Category | Meaning |
|---|---|
| Recurring operational work | Work that happens regularly and can likely be documented as an SOP. |
| Ad hoc support request | Work triggered by a specific request from another person, team or system. |
| Exception handling | Work that happens when the standard path does not apply or fails. |
| Informal workaround | Manual or unofficial work used to complete a task despite gaps in process, system or ownership. |
| Escalation | Work that needs to be passed to another person, manager, team or system owner. |
| Quality control | Checks or safeguards used to maintain data quality, accuracy or consistency. |
| Role gap | Work expected by others or described in role documents but not currently performed. |
| Future capability signal | Work that may be needed in a future operating model but is not current practice. |
