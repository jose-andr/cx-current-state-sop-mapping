# Assumptions Log

## Purpose

This log captures assumptions, uncertainties and unresolved questions identified during current-state SOP mapping.

The purpose is to avoid treating unverified information as fact.

Assumptions should be reviewed, confirmed, corrected or closed as the work progresses.

## How to use this log

Add an entry when:

- something is believed but not yet confirmed
- a process step is unclear
- ownership is uncertain
- a position description does not match actual practice
- a system dependency needs validation
- a decision rule is undocumented
- an exception pathway is described but not yet verified
- a future implication is plausible but not proven

## Assumption status

Use one of the following statuses:

| Status | Meaning |
|---|---|
| Open | Assumption has not been confirmed |
| Confirmed | Assumption has been validated as true |
| Corrected | Assumption was partly true but needed adjustment |
| Rejected | Assumption was found to be incorrect |
| Superseded | Assumption is no longer relevant |
| Converted to issue | Assumption requires follow-up outside this log |

Review this log:

- before using the repository as business case evidence

## Notes

Do not delete rejected assumptions.
